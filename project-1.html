Porduct Testing Station

During summer internship of 2021 I designed and built a product testing station to automate the production process.

Live Link

Project Overview

The current testing method is not a standardised process and requires the operator testing the product to wave the IR sensors and visually confirm that the valves are actuating while also checking that the sensors are actuating in the correct order. The online monitoring system is tested by connecting a laptop and only reading whether there is a signal or not. This project will provide a solution to standardise and automate the testing process. The project will also utilise the products data output ability to record the flow rate and keep a log of which Stepovers have been tested on OnFarms servers. The solution should increase productivity of the production and testing of the Stepovers and provide traceable information on the assemblies for future fault finding.

The proposed solution is to create a jig at one of the stations of the carousal that is fixed in place to the non-rotating portion. The jig will mount two servo motors that will activate the IR sensors on the stepovers within the acceptable range of 0.05 â€“ 2m. A concept system is shown in Figures 1 and 2. A microcontroller will be connected to the board using a split connector so it can monitor the spray sequence, as well as allow a visual check that is to be done by the operator. The servo motors will be controlled using the orange pi, or any other similar microcontrollers that are readily available. Either a touchscreen or an HMI will be used to input the assembly number of each stepover and indicate a successful or an unsuccessful test. The operator at the testing station should be able to build the system, connect the air supply, and press an easily accessible start button or touchscreen (see Figure 3). The micro controller should activate the servos which will start the spray sequence. Information regarding the time of the testing, the operator who is testing will be recorded to the system server. Also, information of the output of the flow meter will be captured during testing if installed. The system will need a wireless or wired internet connection. All components of the system will be sealed or waterproof. Further development of the project can include implementation into the new stock system and barcodes/QR codes can be utilised.
