Project Summary

I was given the job of revamping an existing cow detection algorithm that had its fair share of problems, primarily with accumulative error. My redesign significantly boosted both reliability and accuracy. To analyze the infrared data, I developed a PyQt5-based GUI. This interface streamlined data processing, virtual PLC runs, data labeling, and comparison of algorithmic results with actual cow positions.

The algorithm I worked on employed linear scoring with weighted variables to determine the cow's leg positions. Factors like cow width, adjacency to other cows, rump position, and additional variables were considered. On the technical side, I faced challenges with real-time distance data processing, largely because the simulation relied on pre-processed data. Balancing this load with the PLC constraints was another aspect I had to carefully manage.

The project had some unique constraints; the Python code had to be convertible to structured text for portability to a Codesys PLC. Parameter tuning was another major part of the project, accomplished using GUILD AI. This allowed me to run overnight experiments to identify the most effective variables for minimizing position errors. Commissioning the robot onsite was another interesting challenge, especially since it involved remote operation of a dairy robot in California with minimal internet connectivity. Among other things, I learned valuable lessons in multithreading and memory management across Linux and Windows environments.
